# Cursor AI Setup Portfolio Project

## Overview

This repository documents the first step of my portfolio project. The goal was to set up Cursor, configure AI tools, create a public GitHub repository, and push a README file explaining the process.

## Tools Installed

* Cursor IDE
* Codex model in Cursor
* Claude model in Cursor
* GitHub

## Steps Completed

1. Installed Cursor IDE.
2. Opened Cursor and checked the available AI model settings.
3. Added Codex as add-on in Cursor.
4. Added Claude Code as in Cursor.
5. Created a public GitHub repository.
6. Opened the repository folder in Cursor.
8. Used Cursor’s built-in terminal to initialize Git and manage repository commands.
9. Created a README.md file.
10. Documented the tools installed, steps completed, issues encountered, and solutions.
11. Committed and pushed the project to GitHub.

## Issues I Ran Into


Cursor is a new IDE for me since I am more familar with Visual Studio. The main challenge I encountered during this task was learning how GitHub integration and pushing repositories worked inside Cursor. I was not sure about how to integrate my local repository into GitHub via Cursor’s terminal. Additionally, some tutorials had pictures of the old version of Cursor, making things appear different compared to the software that was currently installed on my computer.


## How I Solved Them

In order to address this problem, I took a look into Cursor’s integrated terminal and compared the workflow to the ones I have encountered using other tools I had previously used. I studied the Git commands thoroughly, confirmed the connection between repositories prior to pushing them, and visited GitHub to make sure that the README file was uploaded and made publicly available. 

I also watched video tutorials about how to add Claude Code and Codex as add-ons in Cursor. 

It turned out to be a bit tricky at first, but once it was all set up, I gained some practice with Cursor.

## What I Learned

I learned how to set up Cursor, configure AI tools, create a public GitHub repository, and document my process in a README file. I also learned that when tools look different from tutorials, it is important to explore the interface, verify what was completed, and clearly explain the solution.
